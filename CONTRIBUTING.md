# Contributing to ValiChord

Thanks for your interest in ValiChord — an open protocol for independent, blind, tamper-evident verification of claims. Contributions are welcome.

A note on pace and expectations: this is an early, small project, currently maintained by one person (with AI assistance). Reviews may take time, and not every contribution will be merged. That's not a reflection on the work — it's about keeping the protocol coherent and trustworthy.

## What's welcome
- Bug reports and fixes
- Documentation improvements and clarifications
- Tests, and reproductions of issues
- New adapters (ingesting other eval/result formats) and integrations
- Thoughtful design discussion, raised as an issue *before* a large pull request

## What we ask
- **Open an issue first** for anything beyond a small fix, so we can agree the approach before you invest time.
- **Keep changes focused** — one concern per pull request.
- **Include tests** where it makes sense, and make sure existing tests pass.
- **Write it so a person can read it.** Clear, genuine, human-understood contributions — not auto-generated bulk. Quality and intelligibility matter more than volume.

## How to contribute
1. Open an issue describing the bug or proposal.
2. Fork the repository and make your change on a branch.
3. Open a pull request referencing the issue. Describe what you changed and why, in plain language.
