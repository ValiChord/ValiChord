Commit-Reveal Protocol Demo

One agent — researcher and validator in a single conductor — walks through a full ValiChord cycle: blind commitment sealed in a private DNA, phase transition detected by DHT poll, public reveal, and final Harmony Record.

Connecting…
0
Connect to conductor
Read app-config.json (written by setup.mjs), then open an authenticated AppWebSocket on the conductor's app interface.
1
Publish validator profile
DNA 3 — Attestation (shared DHT)
Sets institution so that claim_study can enforce the conflict-of-interest rule at the validate() level. Institution must be non-empty — the validator declares affiliation or fails.
2
Submit validation request
DNA 3 — Attestation (shared DHT)
A fresh random ExternalHash is used as the study data hash each run, so repeated runs never collide on "already claimed" or "already attested" guards. num_validators_required = 1 in this demo network; the DNA property minimum_validators is also 1.
3
Claim study
DNA 3 — Attestation (shared DHT)
Validator self-assigns. validate() enforces the COI rule: validator_institution ≠ researcher_institution. Here researcher_institution = "" (independent researcher) so the COI check is bypassed.
4
Create validation task
DNA 2 — Validator Workspace (private, single-agent)
Stores the task assignment as a private entry. Content never leaves this device. The task_hash returned here is passed to seal_private_attestation as a parent reference.
5
Warm up attestation cell init()
DNA 3 — Attestation
A harmless get_current_phase call forces DNA 3's init() to complete before post_commit fires. Without this, DNA 2's post_commit would trigger init() inside a cross-DNA call while holding the cell operation lock — causing a 30-second conductor deadlock.
6
Seal private attestation — Commit phase
DNA 2 — Validator Workspace (private)
Generates a 32-byte nonce, computes commitment_hash = SHA-256(msgpack(attestation) ‖ nonce), writes the private ValidatorPrivateAttestation entry.
post_commit fires asynchronously, cross-calls DNA 3 notify_commitment_sealed, which writes a CommitmentAnchor to the shared DHT. When the CommitmentAnchor count reaches minimum_validators (1), a PhaseMarker(RevealOpen) is also written.
7
Wait for post_commit
Holochain's post_commit runs asynchronously after the source chain write. This pause lets the cross-DNA call complete before we poll the DHT. Phase transitions are driven by DHT state — not by signals, which are send-and-forget with no delivery guarantee.
8
Check protocol phase
DNA 3 — Attestation
get_current_phase reads the latest PhaseMarker entry via the RequestToPhaseMarker link path. Returns "RevealOpen", "Complete", or null (still in commit phase). The protocol gate is purely DHT state — no signal required.
9
Submit attestation — Reveal phase
DNA 3 — Attestation (shared DHT, required_validations = 7)
Publishes the full ValidationAttestation. The coordinator verifies SHA-256(msgpack(attestation) ‖ nonce) == CommitmentAnchor.commitment_hash before writing — preventing adaptive reveals. (Hash check bypassed here: empty issuer = dev mode.)
post_commit fires and calls governance check_and_create_harmony_record.
10
Finalise round in Governance DNA
Calls check_and_create_harmony_record explicitly. Although submit_attestation calls governance synchronously, the new link isn't DHT-visible within that same transaction. Calling after submit_attestation returns ensures the ValidatorToAttestation link is available for quorum reads. Governance assembles the HarmonyRecord + badge.
✓
Retrieve Harmony Record
DNA 4 — Governance (public DHT)
Reads via get_harmony_record(request_ref), an Unrestricted function accessible via HTTP Gateway. Immutable after writing — the permanent public outcome of the validation round.
