ValiChord — Live Demo

Submit a research deposit. ValiChord scans it and shows what validators will see. For this demo, the deposit quality score is used as a proxy verdict — so you can watch the full system work end to end. In production, the outcome sealed on-chain is determined by validators (human or AI) who actually run the code.

Stores the task assignment as a private entry. Content never leaves this device. GDPR-compliant by architecture, not by policy.

Warm up attestation cell
DNA 3 — Attestation
Forces DNA 3's init() to complete before post_commit fires — prevents a cross-DNA deadlock on first run.
6
Seal verdict — Commit phase
DNA 2 — Validator Workspace (private)
Generates a 32-byte nonce, computes commitment_hash = SHA-256(msgpack(verdict) ‖ nonce), writes the private entry. Only the hash goes to the shared network — the verdict itself stays sealed. The validator cannot change it after this point.
7
Wait for network confirmation
Holochain's post_commit runs asynchronously. This pause lets the CommitmentAnchor propagate to the shared DHT before we check phase state.
8
Check protocol phase
DNA 3 — Attestation
Reads the PhaseMarker entry from the DHT. Returns RevealOpen once all validators have committed. The phase gate is pure DHT state — no central coordinator required.
9
Reveal verdict — Reveal phase
DNA 3 — Attestation (shared DHT)
Publishes the full verdict. The network verifies SHA-256(msgpack(verdict) ‖ nonce) == commitment_hash before accepting. If the verdict was changed after sealing, the reveal is rejected.
10
Finalise round in Governance DNA
Calls check_and_create_harmony_record. Governance assembles the HarmonyRecord once the validator quorum is satisfied.

Retrieve Harmony Record
DNA 4 — Governance (public DHT)
Reads via get_harmony_record — an Unrestricted function accessible via HTTP Gateway to anyone. Immutable after writing. The permanent public outcome of this validation round.
